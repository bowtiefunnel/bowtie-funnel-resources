Anatomy of a repurpose

One MCP tool → one Bowtie Funnel skill

Every skill above is a raw MCP capability wrapped four ways: rebranded, turned into a repeatable play, its output published to a shareable labs page, and reused across stages. Here's the pattern on Website Visitor De-anonymizer.

Website Visitor De-anonymizer

Raw capability: deanonymize_contacts → Bowtie Funnel skill · Awareness · Retention

Before — raw MCP

  • Two bare tools: deanonymize_contacts → get_results
  • Output is a private artifact_id inside Verbiflow
  • No positioning, no deliverable, no reuse
  • Invisible to the client

After — Bowtie Funnel skill

  • Rebranded + placed at Awareness & Retention on the bowtie
  • Wrapped in a play: de-anon → enrich → qualify → route
  • Output published to a branded labs URL, not a CSV
  • Reused by aim: new traffic = leads; client domains = account-activity signal
