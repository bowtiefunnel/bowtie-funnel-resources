/prospect-list-builder

Every outbound play starts with a list — and a bad list wastes every send after it.

Prospect List Builder. Give it companies, titles, or a persona; it returns named contacts with the fields you need to reach them.

MIT Claude Code skill Awareness Expansion runs on Growth MCP

The repurpose — raw tool → Bowtie skill

before — raw MCP
  • find_people_at_companies / targeted_search_people / linkedin_search_people
  • Output is a private artifact_id
  • No qualification, no reuse
→
after — Bowtie Funnel skill
  • Rebranded + placed at Awareness & Expansion
  • Wrapped in a play: define ICP → build → hand to enrichment
  • Output published as a shareable list page
  • Reused by aim: cold TAM (Awareness); new buyers in current accounts (Expansion)
ICP→search→named contacts→artifact

How it runs

  1. growth_session_init — auth + workspace.
  2. growth_find_people_at_companies — people at a known company list (or growth_targeted_search_people for a persona).
  3. growth_get_people_at_companies_page — page the results.
  4. Hand the artifact_id to Contact Enrichment and Lead Qualification.

Get this skill

Option A — install the GTM plugin (skill + MCP, everything)

# add the marketplace, then install the plugin
/plugin marketplace add bowtiefunnel/bowtie-funnel-Labs
/plugin install bowtie-gtm@bowtie-funnel

Bundles this skill (and the rest) plus the Growth MCP server, wired up.

Option B — just this skill

# 1) add the Growth MCP once (the engine the skill calls)
claude mcp add --transport stdio growth -- npx -y growth-mcp

# 2) copy this one skill into your Claude Code skills dir
cp -r plugins/bowtie-gtm/skills/prospect-list-builder ~/.claude/skills/

Then invoke it, or just say what you want:

/prospect-list-builder

# or, in plain language —
# "build a list of VP Sales at Series-A fintechs"
# "find people at these 30 companies"
