Agent Skills / data-ops-artifacts

/data-ops-artifacts

GTM data is a relay — each skill hands rows to the next.

Data Ops & Artifacts. It manages the artifacts, joins, and exports that move data between every other skill, so nothing is lost between steps.

MIT Claude Code skill All stages runs on Growth MCP

The repurpose — raw tool → Bowtie skill

before — raw MCP
  • artifacts + csv + clay tools
  • Manual glue between steps
  • Easy to lose provenance
→
after — Bowtie Funnel skill
  • Rebranded as the connective tissue
  • Wrapped in a play: chain skills by `artifact_id`, join sources, export deliverables
  • Spans all seven stages of the bowtie
artifacts→join / export / stage→next skill

How it runs

  1. growth_session_init — auth + workspace.
  2. growth_list_artifacts / growth_get_artifact — find prior rows, jobs, joins.
  3. growth_join_csv_files / growth_join_clay_results_by_domain — merge sources.
  4. growth_clay_export_csv / growth_export_job_csv — export deliverables.
  5. growth_stage_artifact_to_sequence_source — stage rows into a sequence for review.

Get this skill

Option A — install the GTM plugin (skill + MCP, everything)

# add the marketplace, then install the plugin
/plugin marketplace add bowtiefunnel/bowtie-funnel-Labs
/plugin install bowtie-gtm@bowtie-funnel

Bundles this skill (and the rest) plus the Growth MCP server, wired up.

Option B — just this skill

# 1) add the Growth MCP once (the engine the skill calls)
claude mcp add --transport stdio growth -- npx -y growth-mcp

# 2) copy this one skill into your Claude Code skills dir
cp -r plugins/bowtie-gtm/skills/data-ops-artifacts ~/.claude/skills/

Then invoke it, or just say what you want:

/data-ops-artifacts

# or, in plain language —
# "join these two lists by domain"
# "export this artifact to CSV"
